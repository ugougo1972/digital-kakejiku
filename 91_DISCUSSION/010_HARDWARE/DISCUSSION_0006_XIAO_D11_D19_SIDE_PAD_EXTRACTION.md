# DISCUSSION_0006: XIAO D11〜D19側面ランド取り出し・GND・JST・ノイズ対策検討

作成日: 2026-06-25  
対象プロジェクト: digital-kakejiku  
分類: HARDWARE  
状態: RECORDED

---

# 1. 議論の目的

本書は、digital-kakejiku 初号機における XIAO ESP32S3 Plus のGPIO取り出し方針、D11〜D19側面ランドの扱い、GNDの取り方、JST 2ピン接続、ノイズ対策、関連する電源部品変更を記録する。

本議論は、以下の方針変更を受けて整理したものである。

- ePaperドライバーボード装着時、XIAO裏面ランドが使用困難である。
- 1.27mm側面ランドはD11〜D19に相当する。
- D11〜D19側面ランドは使用解禁とする。
- ePaper本体には長めのリボンケーブルが付属しており、ドライバーボード＋XIAOを本体基板近傍へ配置できる。
- D11〜D19は、長距離空中配線ではなく短距離接続を前提に再整理する。

---

# 2. 背景

当初、XIAO ESP32S3 Plus のGPIO不足対策として、裏面ランドの使用を前提としていた。  
しかし、XIAOをePaperドライバーボードへ装着すると、裏面ランドへ物理的にアクセスできないことが判明した。

その後、1.27mm側面ランドがD11〜D19に相当することを確認し、側面ランドを追加GPIO取り出しに利用する方針へ変更した。

さらに、ePaper本体に長めのリボンケーブルが付属していることを確認したため、ePaperドライバーボード＋XIAOを本体基板近傍へ配置できる見込みとなった。

この結果、D11〜D19を20〜30cmの長距離空中配線として扱う必要性は低下した。

---

# 3. 検討した構成

## 3.1 旧案: 裏面ランド使用

```text
XIAO裏面ランド
  ↓
追加GPIO取り出し
```

不採択理由。

- ePaperドライバーボード装着時に裏面ランドへアクセスできない。
- 物理実装が困難である。
- ドキュメント上の「裏面ランド使用可」前提を廃止する必要がある。

## 3.2 旧案: XIAO 2台構成

```text
XIAO-A: ePaper / GAS / 表示
XIAO-B: センサー / UI / RTC / microSD
```

不採択理由。

- MCU間通信が必要になる。
- 電源投入順、時刻同期、ログ統合が複雑化する。
- ファームウェア管理が二重化する。
- 初号機の故障点が増える。

結論。

```text
初号機ではXIAO 1台構成を維持する。
2台構成はフォールバック案としてのみ記録する。
```

## 3.3 採択案: D11〜D19側面ランド取り出し

```text
XIAO D11〜D19側面ランド
  ↓
D11〜D19取り出し用ミニ基板
  ↓
JST 2ピン 信号 + GND
  ↓
撚り線
  ↓
大型本体基板
```

採択理由。

- 裏面ランドを使わずにGPIOを取り出せる。
- XIAO 1台構成を維持できる。
- ePaperドライバーボード＋XIAOを本体基板近傍へ配置できるため、配線長を短縮できる。
- 信号線とGNDをペア化できる。
- GPIO番号をJSTごとに明示できる。

# 4. D11〜D19取り出し用ミニ基板の考え方

## 4.1 役割

D11〜D19取り出し用ミニ基板は、信号取り出し専用とする。

```text
役割:
  - D11〜D19信号線の取り出し
  - XIAO GNDの取り出し
  - 小規模GNDバスの形成
  - JST 2ピン化
```

## 4.2 使途未確定段階で抵抗を載せない理由

D11〜D19の用途は、現時点では未確定である。  
用途が未確定の段階で直列抵抗、プルアップ、プルダウン、RCフィルタを載せると、後から用途に合わない可能性がある。

したがって、ミニ基板は以下に特化する。

```text
信号線取り出し
GND取り出し
JST 2ピン化
```

抵抗類は、用途が確定した後、本体基板側または信号源側で設計する。

## 4.3 将来の抵抗配置の整理

| 信号用途 | 抵抗・部品 | 基本配置 |
|---|---|---|
| GPIO出力 | 直列抵抗 | 信号源側 |
| GPIO入力 | プルアップ/プルダウン | 受け側入力近く |
| SPI CS | プルアップ | 対象デバイス側または本体基板側 |
| SPI SCK/MOSI | 直列抵抗 | XIAO側または信号源側 |
| SPI MISO | 直列抵抗 | microSD等の出力側 |
| 割り込み線 | プルアップ | 受け側近傍 |
| ADC | 分圧抵抗 + RC | ADC入力近傍 |
| I2C | プルアップ | バス全体で1組、3.3Vへ |

# 5. GNDの検討

## 5.1 GNDは信号の戻り道である

信号線は単独で動作するものではなく、必ず戻り電流の経路を持つ。  
信号線とGNDが離れていると、ループ面積が大きくなり、ノイズを拾いやすくなる。

このため、D11〜D19では、信号線とGNDをペア化する方針とした。

## 5.2 ミニ基板上のGNDバス

ミニ基板には、XIAO由来のGNDを取り込み、小規模GNDバスを設ける。

```text
XIAO GND
  ↓
ミニ基板 GNDバス
  ├─ JST-D11 GND
  ├─ JST-D12 GND
  ├─ ...
  └─ JST-D19 GND
```

このGNDは信号用GNDであり、大電流帰路ではない。

## 5.3 本体基板側GND

本体基板側JSTのGNDは、本体基板GNDバスへ直結する。

```text
JST-D11 GND
  ↓
本体基板 GNDバス
```

ただし、以下は避ける。

```text
JST-D11 GND
  ↓
JST-D12 GND
  ↓
SPS30 GND
  ↓
DC-DC GND
```

これはGNDの数珠つなぎであり、信号GNDに電源変動が乗る可能性がある。

推奨は以下である。

```text
各JST GND
  ↓
本体基板 信号用GNDバス
  ↓
本体基板 共通GND
  ↓
電源GND基準点
```

# 6. JST 2ピン方式の検討

## 6.1 採択理由

D11〜D19を1本の多ピンコネクタでまとめる案も考えられる。  
しかし、初号機ではJST 2ピンをGPIOごとに分ける方式を優先する。

理由。

- 各コネクタにD11、D12等のGPIO番号を明示できる。
- 信号線とGNDが常に1組になる。
- テスターでの導通確認が容易である。
- 誤配線時の切り分けが容易である。
- 1信号単位で着脱・交換できる。
- 初号機では小型化より保守性を優先する。

## 6.2 接続ルール

| JST | Pin1 | Pin2 |
|---|---|---|
| JST-D11 | D11 | GND |
| JST-D12 | D12 | GND |
| JST-D13 | D13 | GND |
| JST-D14 | D14 | GND |
| JST-D15 | D15 | GND |
| JST-D16 | D16 | GND |
| JST-D17 | D17 | GND |
| JST-D18 | D18 | GND |
| JST-D19 | D19 | GND |

Pin1は信号、Pin2はGNDに統一する。

## 6.3 ケーブル

各JSTから本体基板までは、信号線＋GNDを撚り線とする。

```text
D11 + GND 撚り線
D12 + GND 撚り線
...
D19 + GND 撚り線
```

この構成により、信号ループ面積を小さくし、ノイズ耐性を高める。

# 7. ノイズ対策の整理

D11〜D19は短距離接続を前提とするが、ノイズ対策は不要にはならない。

| 対策 | 判断 | 備考 |
|---|---|---|
| 信号線＋GNDペア | 採択 | 基本方針 |
| 撚り線 | 採択 | 筐体内配線のノイズ低減 |
| 本体基板近傍配置 | 採択 | 配線短縮 |
| DC-DCから距離確保 | 推奨 | TPS63802、IP5306周辺に注意 |
| SPS30ファン電源線から距離確保 | 推奨 | 5V系の負荷変動に注意 |
| ePaper電源線から距離確保 | 推奨 | 更新時の電流変動に注意 |
| 直列抵抗 | 必要時 | 用途確定後に判断 |
| プルアップ/プルダウン | 必要時 | 入力用途確定後に判断 |
| ADC RCフィルタ | 必要時 | 電圧監視用途確定後に判断 |

# 8. ePaper配置の再整理

当初、ePaperドライバーボードはePaper本体近傍に配置される前提だった。  
その場合、D11〜D19から本体基板まで20〜30cm程度の空中配線になる可能性があった。

しかし、ePaper本体には長めのリボンケーブルが付属しているため、以下が可能となった。

```text
ePaper本体
  ↓ 付属リボンケーブル
ePaperドライバーボード + XIAO
  ↓ 短距離接続
大型本体基板
```

この変更により、D11〜D19を長距離ハーネスではなく、短距離の基板間接続として扱える。

注意点。

- ePaperリボンケーブルを無理に折り曲げない。
- リボンケーブルに機械的ストレスをかけない。
- ePaper更新時の電源変動を5V/3.3V設計で確認する。
- SPI共有制御の論理方針は変更しない。

# 9. I2S再検討

現行文書では、ICS-43434のI2S信号としてGPIO39、GPIO40、GPIO41を使用する前提が残っている。  
これは裏面ランド使用前提の割当である。

裏面ランドを使用禁止としたため、I2S割当は再検討が必要である。

未確定事項。

- ICS-43434を初号機で維持するか
- I2S信号をD11〜D19へ割り当てるか
- I2S信号を別GPIOへ割り当てるか
- マイク配置をXIAO近傍へ寄せられるか
- 音環境観測をPhase後半へ延期するか

現時点では、I2Sは高速寄りのクロック信号を含むため、D11〜D19への割当は慎重に扱う。

# 10. SPS30電源検討

SPS30の動作電圧は3.5V〜5.0Vで確定した。  
このため、SPS30は5V BUS負荷として扱う。

```text
IP5306
  ↓
5V BUS
  ├─ SPS30
  └─ TPS63802
       ↓
     3.3V BUS
```

注意点。

- SPS30を3.3V BUSに接続しない。
- SPS30のGND帰路は大電流側として太めに本体基板GNDへ戻す。
- D11〜D19用GNDをSPS30の帰路に使わない。
- I2Cプルアップ先は3.3Vに統一する。

# 11. DMG3415U採択検討

当初、逆流防止Pch MOSFETとしてDMG2305UX-13を採択していた。  
手元在庫としてDMG3415Uがあるため、初号機ではDMG3415Uを採択する。

| 項目 | 判断 |
|---|---|
| 初号機採用品 | DMG3415U |
| 代替候補 | DMG2305UX-13 |
| 用途 | 5V BUS逆流防止 |
| ヒートシンク | 原則不要 |
| 放熱 | 銅箔面積と実測で確認 |
| 長期BOM | 再評価対象 |

実測対象。

- SPS30動作時
- ePaper更新時
- Wi-Fi送信時
- BATTERY_MODE時
- 5V BUS電圧降下
- MOSFET表面温度

# 12. 本議論で得た設計ルール

```text
1. 裏面ランドは使用禁止。
2. 1.27mm側面ランド、すなわちD11〜D19は使用解禁。
3. D11〜D19は長距離空中配線にしない。
4. ePaperドライバーボード＋XIAOは本体基板近傍へ配置する。
5. D11〜D19取り出し用ミニ基板は信号線＋GND取り出し専用とする。
6. ミニ基板には小規模GNDバスを設ける。
7. D11〜D19はJST 2ピンで信号＋GNDペア化する。
8. JSTケーブルは信号線＋GNDの撚り線とする。
9. 本体基板側JSTのGNDは本体基板GNDバスへ直結する。
10. D11〜D19用GNDへ大電流帰路を流さない。
11. 抵抗類は用途確定後に本体基板側または信号源側で設計する。
12. SPS30は5V BUS負荷とする。
13. DMG3415Uを初号機の逆流防止Pch MOSFETとして採択する。
```
