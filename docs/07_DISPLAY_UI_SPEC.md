# 07_DISPLAY_UI_SPEC.md

最終更新: 2026-06-03

---

## 1. 概要

本書は digital-kakejiku の 7.5inch 800×480 E-Paper 表示仕様を定義する。

本機は据置型・常時稼働の環境観測器であり、頻繁なアニメーションや高FPS表示ではなく、静的で読みやすい表示を重視する。

E-Paperの更新回数を抑え、観測情報、状態情報、診断情報を段階的に表示する。

## 4. UI基本方針

- 静的表示を基本とする
- 余白を確保する
- 情報を詰め込みすぎない
- 更新頻度を抑える
- 観測値は一目で読める大きさを優先する
- 診断情報は通常画面から分離する
- AI表示はPhase2以降の将来拡張とする

---


## 4.1 前面・背面の責務分離

査読指摘を反映し、前面E-Paperと背面OLEDの責務を明確に分離する。

| 表示面 | デバイス | 主用途 | 担当Manager |
|---|---|---|---|
| 前面 | 7.5inch E-Paper | 日常表示、日めくり、環境情報、暦情報、今日の詩 | DisplayManager |
| 背面 | OLED 128×96 | 設定、診断、保守、手動操作 | UIManager / DiagnosticManager |

前面E-Paperは通常操作を要求しない表示専用とする。

背面OLEDは操作・保守専用とし、通常閲覧用の主要表示には使わない。

## 5. 入力装置

| 入力 | 内容 |
|---|---|
| ロータリーエンコーダ回転 | 画面切替、項目選択 |
| プッシュスイッチ | 決定、詳細表示、メニュー遷移 |

本プロジェクトでは、RGB LED付きロータリーエンコーダは採択しない。

入力装置は、RGB無しの水平ダイヤル式・押下スイッチ付きロータリーエンコーダを前提とする。

| 入力 | 用途 |

| ロータリーエンコーダ回転 | 項目選択 |
| ロータリーエンコーダ押下 | 決定 |

3ポジションダイヤルは背面OLEDの画面群切替を担当する。

### 7.4 暦・詩表示項目

2026-06-19時点の採択事項として、HOME画面には以下を追加する。

| 項目 | データ元 | 失敗時 |
|---|---|---|
| 六曜 | calendar_master | 取得できません |
| 二十四節気 | calendar_master | 取得できません |
| 七十二候 | calendar_master | 取得できません |
| 今日の詩 | poem_cache | 取得できません |

Calendar情報はGAS側で生成する。ESP32側では暦生成を行わない。

今日の詩は `poem_cache` を表示する。表示時にGemini APIを再呼び出ししない。

### 10.4 Poem表示への整理

従来のAI画面は、2026-06-19時点ではPoem Subsystemの「今日の詩」表示へ整理する。

- エラー時の代替詩生成

AI画面という名称は将来拡張として残すが、初期実装では `poem_cache` 表示を正とする。

## 13. 表示更新方針

### 13.1 基本方針

- 部分更新を優先する
- 全面更新は必要時のみ実施する
- microSDアクセス中はE-Paper更新を行わない
- 停電時は表示更新よりログ保存を優先する

### 13.2 更新タイミング

| 条件 | 更新 |
|---|---|
| 時刻表示の更新 | 部分更新 |
| センサー値の大きな変化 | 部分更新 |
| 画面切替 | 全面更新または必要範囲更新 |
| エラー発生 | 必要時のみ更新 |
| 停電検知 | 最小限の更新 |
| 復電検知 | 状態表示更新 |

---

## 14. SPI共有に関する制約

E-Paper と microSD は SPI バスを共有する。

そのため、DisplayManager は StorageManager と同時に SPI を使用しない。

| 状態 | 方針 |
|---|---|
| microSD書込中 | E-Paper更新を待機 |
| E-Paper更新中 | microSD書込を待機 |
| 停電時 | microSD保存を優先 |
| 通常時 | SystemManagerが順序制御 |

---


### 14.1 SPI ResourceManager参照

SPI共有制御の詳細は `09_SPI_RESOURCE_CONTROL.md` を正とする。

表示更新要求がmicroSD書込と競合する場合は、ログ保存を優先する。

E-Paper更新は再試行可能な処理として扱い、停電時およびStorageManager処理中は抑制する。

## 15. 電源状態表示

| 状態 | 表示内容 |
|---|---|
| USB給電 | 通常稼働 |
| BATTERY_MODE | バッテリー動作中 |
| 低電圧 | 警告表示 |
| 復電 | 通常稼働へ復帰 |

停電時は表示更新を最小限にする。

## 16. エラー表示

| エラー | 表示方針 |
|---|---|
| SENSOR_ERROR | 該当センサーを警告表示 |
| STORAGE_ERROR | SD異常を明示 |
| NETWORK_ERROR | 通信異常を小さく表示 |
| RTC_ERROR | 時刻異常を表示 |
| DISPLAY_ERROR | ログ記録を優先 |
| POWER_ERROR | 電源警告を優先表示 |

表示不能時はログ保存を正とする。

---


### 16.1 Calendar / Poemエラー表示の寿命

査読指摘を反映し、Calendar/Poem失敗時の表示継続方針を定義する。

| 対象 | 表示 | 継続条件 | 再試行方針 |
|---|---|---|---|
| Calendar取得失敗 | 取得できません | 次回Calendar更新成功まで | GAS側スケジュールまたは手動更新 |
| Poem生成失敗 | 取得できません | 当日分のpoem_cache更新成功まで | GAS側Poemリトライ。表示時再生成は禁止 |
| source_config失敗 | 取得できません | 情報源回復まで | error_log記録後、次回ジョブで再試行 |
| Gemini API失敗 | 取得できません | poem_cacheがokになるまで | Poem Subsystem側で制御 |

E-Paper側は、該当領域のみ部分更新する方針とする。ただし、部分更新可否は実機評価で確認する。

表示側で代替文、暦、詩を生成してはならない。
